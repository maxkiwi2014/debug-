1. 實驗目標

本專題目標是建立一個可分析 MP3/WAV 與 YouTube 音訊的 AI 網站，透過 YAMNet 模型辨識樂器與人聲，並顯示信心度、可信度與明顯出現秒數。

2. Debug 總覽

問題 主要原因 解決結果
樂器存在但沒有被顯示 早期版本只顯示 Top 3，部分樂器可能被排除。 移除 Top 3 限制，顯示所有超過 5% 門檻的項目。
鼓與 Bass 類別太分散 YAMNet 會輸出 Drum kit、Snare drum、Hi-hat 等細項。 建立合併分類，將鼓類統一成「鼓」、Bass 類統一成「Bass」。
鼓的信心度被低估 鼓分數被分散在多個原始類別中。 將合併類別分數加總，最高封頂 100%。
YouTube 音訊無法分析 下載格式可能是 mp4 或 webm，librosa 不一定能讀取。 加入 imageio-ffmpeg，下載後轉成 wav。
同一首歌分析結果不同 YouTube 搜尋可能找到不同版本，且早期固定只分析前 90 秒。 新增 YouTube 連結模式與分析長度設定。
網站無法開啟 Streamlit 伺服器沒有啟動。 重新啟動 Streamlit 後，localhost 可正常連線。

3. 詳細 Debug 紀錄

Debug 1

樂器存在但沒有被顯示

問題描述：測試音檔時，使用者認為有鋼琴、鼓或 Bass，但網站沒有顯示出來。

原因分析：早期版本只顯示 Top 3，若樂器排名在第 4 名以後，或低於 5% 門檻，就不會出現在畫面上。

修改前 
if len(visible_results) == 3:
    break
修改後 
if score < CONFIDENCE_THRESHOLD:
    continue

visible_results.append((display_name, score, sources, active_times))
結果：移除 Top 3 限制後，所有超過 5% 信心度門檻的樂器與人聲都會被列出。
Debug 2

鼓與 Bass 類別太分散

問題描述：YAMNet 會把鼓拆成 Drum kit、Snare drum、Hi-hat、Bass drum 等細項，導致結果太雜。

原因分析：YAMNet 是通用聲音分類模型，會輸出 AudioSet 原始英文類別，不一定符合一般使用者的音樂理解。

修改前 
# 早期可能直接顯示模型原始類別
Drum kit
Snare drum
Hi-hat
Bass drum
Bass guitar
修改後 
DRUM_CLASSES = {
    "Bass drum",
    "Cymbal",
    "Drum",
    "Drum kit",
    "Hi-hat",
    "Percussion",
    "Snare drum",
}

BASS_CLASSES = {
    "Bass guitar",
    "Double bass",
}

def get_display_name(class_name):
    if class_name in DRUM_CLASSES:
        return "🥁 鼓"

    if class_name in BASS_CLASSES:
        return "🎸 Bass"
結果：鼓類統一顯示為「🥁 鼓」，Bass 類統一顯示為「🎸 Bass」，結果更容易理解。
Debug 3

鼓的信心度被低估

問題描述：使用者認為鼓很明顯，但系統顯示的鼓分數偏低。

原因分析：鼓的分數被分散在 Drum kit、Snare drum、Hi-hat、Cymbal 等多個類別中，單看任何一個類別都可能不高。

修改前 
# 修改前只取最高的一個來源類別
score = sources[0][1]
修改後 
# 修改後將合併類別的來源分數加總，最高封頂 100%
score = min(sum(source_score for _, source_score, _ in sources), 1.0)
結果：鼓、Bass、人聲這類合併項目會用整體分數呈現，較符合實際聽感。
Debug 4

YouTube 音訊無法分析

問題描述：加入 YouTube 搜尋後，網站顯示「YouTube 搜尋或分析時發生錯誤」。

原因分析：yt-dlp 下載的音訊可能是 mp4 或 webm，librosa 不一定能穩定讀取。

修改前 
ydl_opts = {
    "format": "bestaudio/best",
    "outtmpl": f"{YOUTUBE_AUDIO_PREFIX}.%(ext)s",
}
修改後 
ydl_opts = {
    "format": "bestaudio[ext=m4a]/bestaudio[ext=mp3]/bestaudio/best",
    "outtmpl": f"{YOUTUBE_AUDIO_PREFIX}.%(ext)s",
    "ffmpeg_location": imageio_ffmpeg.get_ffmpeg_exe(),
    "postprocessors": [
        {
            "key": "FFmpegExtractAudio",
            "preferredcodec": "wav",
        }
    ],
}
結果：YouTube 音訊會先轉成 wav，再進入 YAMNet 分析流程，分析穩定性提升。
Debug 5

同一首歌分析結果不同

問題描述：同樣輸入「私奔到月球」，YouTube 分析結果與上傳音檔不同。

原因分析：YouTube 搜尋可能找到 MV、Live、Cover 或歌詞版，而且早期固定只分析前 90 秒。

修改前 
# 修改前：只靠搜尋第一個結果，且固定分析前 90 秒
search_result = ydl.extract_info(f"ytsearch1:{search_text}", download=True)
MAX_ANALYSIS_SECONDS = 90
修改後 
analysis_duration = st.selectbox(
    "選擇分析長度",
    ["90 秒", "30 秒", "全曲"],
    index=0,
)

youtube_input_mode = st.radio(
    "YouTube 來源方式",
    ["搜尋名稱", "貼上 YouTube 連結"],
    horizontal=True,
)
結果：使用者可以貼指定 YouTube 連結，也能選擇 30 秒、90 秒或全曲分析。
Debug 6

網站無法開啟

問題描述：瀏覽器出現 localhost 拒絕連線，無法開啟網站。

原因分析：Streamlit 伺服器沒有啟動，並不是 app.py 語法錯誤。

修改前 
# 瀏覽器開啟：
http://localhost:8501

# 但沒有 Streamlit process 在背景執行
修改後 
C:\Users\maxki\AppData\Local\Programs\Python\Python313\python.exe -m streamlit run app.py
結果：重新啟動 Streamlit 後，localhost:8501 可以正常連線。

4. Debug 工具設計

除了修正主要問題，專題也加入原始分數除錯區，讓使用者能看到 YAMNet 背後的原始類別分數。

DEBUG_CLASSES = [
    "Speech",
    "Singing",
    "Vocal music",
    "Bass guitar",
    "Double bass",
    "Drum",
    "Drum kit",
    "Snare drum",
    "Hi-hat",
    "Cymbal",
    "Percussion",
]

def get_debug_scores(peak_scores, class_names):
    score_by_class = {
        class_name: float(peak_scores[index])
        for index, class_name in enumerate(class_names)
    }

    return [
        (class_name, score_by_class[class_name])
        for class_name in DEBUG_CLASSES
        if class_name in score_by_class
    ]

本專題的 Debug 過程顯示，AI 應用不只是把模型接上網站，還需要處理資料格式、分類轉譯、錯誤提示與使用者理解問題。透過這些修正，網站從單純辨識工具，逐步變成可以解釋結果、讓使用者驗證判斷依據的音樂分析輔助平台。
